# Supabase Database Setup Guide

To resolve the "table not found" errors in the backend, you need to create the required tables in your Supabase project.

## SQL Setup Instructions

1.  Log in to your [Supabase Dashboard](https://supabase.com/dashboard).
2.  Select your project.
3.  Click on the **SQL Editor** in the left sidebar.
4.  Click **New Query**.
5.  Copy and paste the following SQL script into the editor:

```sql
-- Create system_metrics table for API monitoring
CREATE TABLE IF NOT EXISTS public.system_metrics (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    path TEXT NOT NULL,
    method TEXT NOT NULL,
    status_code INTEGER NOT NULL,
    latency FLOAT8 NOT NULL,
    cache_hit BOOLEAN DEFAULT FALSE
);

-- Create activity_logs table for audit trails
CREATE TABLE IF NOT EXISTS public.activity_logs (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    user_id TEXT,
    user_type TEXT,
    activity_type TEXT,
    description TEXT,
    ip_address TEXT,
    user_agent TEXT,
    metadata JSONB DEFAULT '{}'::jsonb
);

-- Enable Row Level Security (RLS)
ALTER TABLE public.system_metrics ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.activity_logs ENABLE ROW LEVEL SECURITY;

-- Create basic access policies (Allow all for service role/authenticated users)
-- Adjust these policies later based on your security requirements
CREATE POLICY "Enable all for everyone" ON public.system_metrics FOR ALL USING (true);
CREATE POLICY "Enable all for everyone" ON public.activity_logs FOR ALL USING (true);
```

6.  Click **Run**.

Once the command finishes successfully, the tables will be created, and the backend error console will stop showing the "table not found" error.

## Verifying the Setup

After running the SQL:
1.  Navigate to the **Table Editor** in Supabase to confirm `system_metrics` and `activity_logs` exist.
2.  Restart your backend server.
3.  Use the application normally (e.g., browse some pages).
4.  Refresh the `system_metrics` table; you should see new rows appearing automatically.
